# UI/UX Mockup Prompt untuk Lovable - Flutter Responsive Design

## Context & Overview

Kamu sedang membuat aplikasi POS terintegrasi untuk toko suku cadang motor dengan modul keuangan & pajak. Aplikasi ini dibangun dengan **Flutter untuk mobile dan web dengan responsive design**.

**Key Requirement:** Desain harus optimal untuk mobile (primary) dan responsif untuk tablet/desktop tanpa perlu redesign terpisah.

**Target Users:**

- Admin/Owner (Bos): Akses penuh, fokus pada dashboard finansial
- Cashier: Akses terbatas, fokus pada transaksi penjualan

**Design Principles (Flutter-specific):**

- Mobile-first approach (optimize untuk 360px-480px width)
- Responsive layout menggunakan Flutter's responsive patterns
- Single codebase untuk mobile, tablet, desktop
- Touch-friendly UI (minimum 48x48dp tap targets)
- Efficient use of screen space
- Consistent Material Design 3 atau Cupertino style
- Dark mode support

---

## Responsive Breakpoints & Layout Strategy

### Breakpoints (Flutter MediaQuery)

```
Mobile:    0 - 599px   (phones)
Tablet:    600 - 1199px (tablets)
Desktop:   1200px+     (desktop/web)
```

### Layout Patterns

#### Mobile (0-599px)

- Single column layout
- Bottom navigation bar (5 items max)
- Full-width cards
- Stacked forms
- Drawer untuk secondary menu (optional)

#### Tablet (600-1199px)

- 2-column layout (sidebar + content)
- Top navigation bar
- Wider cards dengan padding
- Side-by-side forms

#### Desktop (1200px+)

- 3-column layout (sidebar + main + detail panel)
- Top navigation bar
- Optimized spacing
- Multi-panel view

### Key Elements

- Header dengan greeting & sync status
- Real-time Profit Widget (expandable breakdown)
- Pajak Berjalan progress indicator
- Quick stats cards (horizontal scroll di mobile)
- Trend grafik (responsive chart)
- Breakdown per buyer tier
- Bottom navigation (mobile) / Top navigation (tablet/desktop)

### Flutter Implementation Notes

- Use `LayoutBuilder` untuk responsive layout
- `MediaQuery.of(context).size.width` untuk breakpoints
- `SingleChildScrollView` + `Column` untuk mobile
- `Row` + `Expanded` untuk tablet/desktop
- `ResponsiveGridView` atau `GridView.builder` untuk cards
- `fl_chart` package untuk grafik
- `Wrap` untuk horizontal scroll stats

## General Flutter Responsive Guidelines

### Responsive Breakpoints

```dart
const double mobileWidth = 600;
const double tabletWidth = 1200;

bool isMobile(BuildContext context) =>
  MediaQuery.of(context).size.width < mobileWidth;

bool isTablet(BuildContext context) =>
  MediaQuery.of(context).size.width >= mobileWidth &&
  MediaQuery.of(context).size.width < tabletWidth;

bool isDesktop(BuildContext context) =>
  MediaQuery.of(context).size.width >= tabletWidth;
```

### Responsive Widgets

```dart
// Use LayoutBuilder untuk responsive layout
LayoutBuilder(
  builder: (context, constraints) {
    if (constraints.maxWidth < 600) {
      return MobileLayout();
    } else if (constraints.maxWidth < 1200) {
      return TabletLayout();
    } else {
      return DesktopLayout();
    }
  },
)

// Use MediaQuery untuk responsive values
double padding = MediaQuery.of(context).size.width < 600 ? 16 : 24;

// Use Expanded & Flexible untuk responsive sizing
Row(
  children: [
    Expanded(flex: 1, child: Widget1()),
    Expanded(flex: 2, child: Widget2()),
  ],
)

// Use GridView.builder untuk responsive grid
GridView.builder(
  gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
    crossAxisCount: MediaQuery.of(context).size.width < 600 ? 1 : 2,
  ),
  itemBuilder: (context, index) => ProductCard(),
)
```

### Touch-Friendly Design

- Minimum tap target: 48x48dp
- Padding between buttons: 8-16dp
- Use `GestureDetector` atau `InkWell` untuk interactive elements
- Provide visual feedback (ripple effect, color change)

### Performance Optimization

- Use `const` constructors
- Use `ListView.builder` untuk long lists
- Use `RepaintBoundary` untuk expensive widgets
- Lazy load images dengan `Image.network` + `cacheHeight`
- Use `SingleChildScrollView` dengan caution (prefer `ListView`)

### Navigation Pattern

- Mobile: Bottom navigation bar (5 items max)
- Tablet: Sidebar + top navigation
- Desktop: Sidebar + top navigation + detail panel

### Color & Typography

- Use Material Design 3 color system
- Consistent typography scale
- Dark mode support dengan `ThemeData`
- High contrast untuk accessibility
